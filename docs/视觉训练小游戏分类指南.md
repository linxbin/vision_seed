## 目的

这份文档用于约束 VisionSeed 的训练分类、选题方向和实现口径。  
它不是实时进度面板；当前实际接入状态以 [core/game_registry.py](/C:/workspace/python/vision_seed/core/game_registry.py) 为准。

## 顶层分类

当前项目稳定保留六个顶层分类：

1. `accommodation`
2. `simultaneous`
3. `fusion`
4. `suppression`
5. `stereopsis`
6. `amblyopia`

除非产品明确要求，不调整这六类的 taxonomy。

## 分类原则

### 1. 一个游戏只服务一个主训练目标

分类以该游戏最核心的训练目标为准，不做多分类重复挂载。

### 2. 训练目标优先于经典玩法复刻

项目不是经典街机合集。  
即使借用熟悉玩法，也应优先服务视觉训练目标，而不是机械照搬原作规则。

### 3. 会话要短，反馈要清楚

训练游戏应尽量满足：

- 短回合或短会话
- 立即反馈
- 简明成功/失败逻辑
- 明确下一步引导

## UI / UX 口径

### 平台页

平台页统一使用亮色主题，见 [core/ui_theme.py](/C:/workspace/python/vision_seed/core/ui_theme.py)。  
主菜单、分类页、系统设置、首次引导、授权页应保持一致语言，不回退到暗色全屏风格。

### 训练 HUD

训练页优先遵循：

- 左上：进度或模式信息
- 顶部中间：倒计时、主分数或主状态
- 右上：返回或次要状态

避免芯片、提示和帮助文案彼此遮挡。

### 儿童向交互

默认键位口径：

- 方向键：移动或导航
- `Space`：主要动作
- `Enter`：继续或确认
- `Esc`：返回

鼠标可以保留，但不应成为唯一可用路径。

## 红蓝模式口径

凡涉及红蓝眼镜模式的游戏，优先复用：

- [games/common/anaglyph.py](/C:/workspace/python/vision_seed/games/common/anaglyph.py)
- [games/simultaneous/eye_find_patterns](/C:/workspace/python/vision_seed/games/simultaneous/eye_find_patterns)

统一支持：

- `left_red_right_blue`
- `left_blue_right_red`

当前平台规则：

- `simultaneous`
- `fusion`
- `suppression`
- `stereopsis`

这四个分类当前统一为红蓝眼镜训练，不再保留裸眼模式入口。  
`accommodation` 和 `amblyopia` 继续保留普通模式训练。

建议规则：

- 训练关键元素使用强红/强蓝
- 背景和帮助元素保持更中性
- 不为单个游戏重新发明一套红蓝渲染体系

## 架构口径

### 全局场景与游戏场景分离

`scenes/` 只放平台级页面。  
游戏私有页面放在 `games/<category>/<game>/scenes/`。

### 游戏接入统一走协议

新游戏通过 `GameDescriptor` 接入，不绕开：

- [core/game_contract.py](/C:/workspace/python/vision_seed/core/game_contract.py)
- [core/game_registry.py](/C:/workspace/python/vision_seed/core/game_registry.py)
- [scenes/game_host_scene.py](/C:/workspace/python/vision_seed/scenes/game_host_scene.py)

### 记录按 `game_id` 隔离

训练数据应始终保留 `game_id` 命名空间，不把多游戏历史混成一套。
